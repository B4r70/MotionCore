# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha:project]** `/sync-gbrain` imports new pages but leaves them UNEMBEDDED (runs `--stale` internally → "Embedded 0 chunks"). Always follow a sync with `gbrain embed --all` in a terminal with Claude Code closed (PGLite lock free).
- **[pattern:project]** Compile-check headlessly without opening Xcode: `xcodebuild -project MotionCore.xcodeproj -scheme MotionCore -destination 'generic/platform=iOS Simulator' -configuration Debug build CODE_SIGNING_ALLOWED=NO`. Builds fine without code-signing and ends in `** BUILD SUCCEEDED **`.
- **[gotcha:project]** A background `xcodebuild` task can report completion with **exit code 0 while the build actually FAILED**. The task-notification status is not reliable — always grep the output file for `** BUILD SUCCEEDED **` (or "The following build commands failed") before treating a compile-check as green.
- **[env:project]** `origin` has TWO push URLs (Forgejo `git.barto.cloud:11022/barto/MotionCore` + GitHub `B4r70/MotionCore`); a single `git push` updates both remotes. PRs are NOT mirrored — open them separately: `gh pr create --repo B4r70/MotionCore` for GitHub (fully authed) and `tea` for Forgejo. The default branch on both is `main`.
- **[gotcha:project]** Forgejo PRs via `tea`: passing a repo slug (`--repo barto/MotionCore` or a 3-part slug) fails with "local repository required". The working invocation from inside the repo is `tea pr create --remote origin --login barto.cloud --head <branch> --base main --title ... --description ...` — discover the login from the remote, don't pass `--repo`.
- **[gotcha:project]** `git switch`/`checkout` to another branch frequently aborts with "local changes to CLAUDE.md would be overwritten" because Caliber/gbrain keep CLAUDE.md modified. Stash just that file first: `git stash push -m tmp -- CLAUDE.md`, switch, commit the intended work, then pop it back — do NOT stash the whole working tree (the rest of the in-flight changes must travel with the switch).
- **[pattern:project]** Watch lifecycle commands that MUST NOT be lost (discard/stop) need belt-and-suspenders delivery: `sendMessage` (fast, only when reachable) PLUS `transferUserInfo` (guaranteed FIFO background delivery) on every send, regardless of reachability. `sendLifecycleMessage`'s `guard isReachable else return` silently drops — use a `sendGuaranteedLifecycle` helper instead. Tag each command with a `lifecycleCommandID` (UUID string) and dedup on the Watch (bounded `processedCommandIDs`) because the double-channel can deliver the same command twice.
- **[pattern:project]** `updateApplicationContext([desiredHealthState: …])` is the reconcile-on-wake anchor for the Watch (persistent, latest-only, read on every activation/scenePhase `.active`). The Watch branches on it: `.discarded`→discard, `.finished`→save, `.active`→leave running. `finished` and `discarded` MUST be SEPARATE enum values — reconcile treats them oppositely (save vs discard), so a dropped finish must not be reinterpreted as a discard.
- **[gotcha:project]** Do NOT push a desired-health-state in `sendIdleState()` (or any path that fires right after a terminal command) — it would clobber the just-set `.finished`/`.discarded` and make the Watch's reconcile mis-handle a dropped command. Desired-state stays at the last terminal value until the next `.active`.
- **[gotcha:project]** Watch Nested-ObservableObject-Falle: `WatchSessionManager` (äußeres `ObservableObject`) hält das live `WatchWorkoutManager` (inneres `ObservableObject` mit `@Published` HR/Kalorien). SwiftUI propagiert die inneren `@Published`-Änderungen NICHT. Lösung: inneres `objectWillChange` über ein `didSet` auf der `workoutManager`-Property nach außen weiterreichen (NICHT als einmalige Init-Subscription), weil `workoutManager` bei Start/Self-Healing/Discard mehrfach neu zugewiesen wird.
- **[gotcha:project]** Watch Teardown-Race: Discard läuft `discardSession()` → `endSession()` setzt `isPaused = true`, und `onChange(of: isPaused)` feuert einen DEFERRED `sendState()` → ein nachlaufender `[workoutState: paused]`-Push trifft NACH der Idle-Message ein. Ein Self-Healing-Guard, der bei `.idle` gelöscht wird, ist vor diesem Trailing-Push schon zurückgesetzt → Phantom-HKWorkout. Zusätzlich den Self-Healing-Guard um `&& lastDesiredHealthState == .active` erweitern, damit ein verspäteter active-Push keine verworfene Session reaktiviert.
- **[pattern:project]** Vor dem Commit bei einem Working-Tree voller fremder Änderungen (`.caliber/`, `.claude/`, gstack-Dateien) zuerst den Feature-Branch anlegen, dann NUR die zu diesem Change gehörenden Dateien explizit `git add`-en — niemals `git add -A`/`.`. Bei gemischten Working-Trees in getrennten Commits splitten (z.B. Tooling/Config separat von Feature-Code).
- **[convention:project]** `.caliber/` (lokale Caliber-Runtime: score-history, learning state, cache) und `.design-review-shots/` (lokale Screenshots) gehören in `.gitignore` — niemals committen. `.mcp.json`, `.caliberignore`, `CALIBER_LEARNINGS.md` und `.claude/` (Hooks, Skills, Commands, Reviews) WERDEN getrackt.
- **[gotcha:project]** Per Cherry-Pick (nicht Merge) in `main` gelandete Branches haben KEINE Ancestor-Beziehung zu `main` → `git branch -d` verweigert sie als "not merged". Inhalt zuerst mit `git diff --stat <branch> origin/main -- <dateien>` prüfen (leer = identisch = sicher), dann mit `git branch -D` löschen. Reflog hält sie ~30 Tage.
- **[pattern:project]** Temporäre On-Device-Debug-Marker (z.B. `os.Logger`-Probes zum Verifizieren von WCSession-Background-Delivery): jede eingefügte Zeile mit `// TEMP PROBE (<step>)` taggen UND die Log-Message mit einem eindeutigen String/Emoji (`🔬PROBE …`) prefixen → trivial per Console.app-Suche findbar und per grep+delete wieder entfernbar. `.notice` statt `.log` verwenden, damit sie ohne Debug-Level sichtbar sind.
- **[convention:project]** Weicht ein Fix bewusst vom „Konkreten Fix" einer Review-Datei ab, den `**Status:**`-Marker des Findings mit ✅ + Datum + Begründung patchen UND dem Sub-Agent explizit „weiche vom literalen Fix ab"-Anweisungen geben.
- **[gotcha:project]** `gbrain` commands (`embed`, `init`, subcommand `--help`) abort with `GBrain: Timed out waiting for PGLite lock` when a background `gbrain serve` holds `~/.gbrain/.gbrain-lock`. Claude Code respawns `gbrain serve` within seconds after `pkill`, so the reliable fix is to run these CLI ops in a terminal with the Claude Code session CLOSED. Locate via `ps aux | grep "gbrain serve"` / `pkill -f "bun.*gbrain serve"`.
- **[gotcha:project]** `gbrain embed --stale` silently embeds nothing (empty output) while `~/.gbrain/config.json` has `embedding_provider: null`, or after a failed (401) embed run. Use `gbrain embed --all` to force all chunks. Keyword/full-text `gbrain search` still works; semantic ranking + `code-def`/`code-refs`/`code-callers` need a provider — source `~/.config/secrets/openai.env` (`set -a; . ~/.config/secrets/openai.env; set +a`) before running.
- **[gotcha:project]** gbrain code-symbol search is unavailable: installed gbrain **0.18.2** has no `reindex-code`, so `/sync-gbrain --full` errors on its code stage; needs gbrain ≥0.20.0. Markdown/memory stages still run. Use Grep for exact Swift symbols.
- **[gotcha:project]** `/sync-gbrain` imports new pages but leaves them UNEMBEDDED (runs `--stale` → "Embedded 0 chunks"). Always follow with `gbrain embed --all` in a terminal with Claude Code closed.
- **[pattern:project]** Native iPhone screenshots for design review arrive as `.HEIC` (AirDropped into `.design-review-shots/`). Convert to PNG before analysis: `sips -s format png in.HEIC --out out.png`. Real device resolution is 1206x2622.
- **[gotcha:project]** gstack `/ios-qa` and `/ios-design-review` DebugBridge expects an SPM `Package.swift` host, but MotionCore is a pure Xcode project (`IPHONEOS_DEPLOYMENT_TARGET = 26.0`). Wiring the bridge means adding a *local* SPM package via the Xcode GUI — editing `project.pbxproj` by hand is fragile.
- **[gotcha:project]** To make a file from a synchronized folder compile in an ADDITIONAL target, add its repo-relative path to that target's `membershipExceptions` inside the `PBXFileSystemSynchronizedBuildFileExceptionSet` block in `project.pbxproj` — do NOT add classic `PBXBuildFile`/`PBXSourcesBuildPhase` entries (this project has none). Files already in two targets via this mechanism (e.g. `WatchHealthDataTypes.swift`, `WatchMessageKeys.swift`) expose new symbols to both targets with no extra project edit.
- **[gotcha:project]** After ANY hand-edit of `project.pbxproj`, run `plutil -lint <path>/project.pbxproj` — must print `OK`. If it doesn't parse, revert; a corrupted project file is worse than a skipped change.
- **[pattern:project]** Compile-check headlessly without opening Xcode: `xcodebuild -project MotionCore.xcodeproj -scheme MotionCore -destination 'generic/platform=iOS Simulator' -configuration Debug build CODE_SIGNING_ALLOWED=NO` (copy `MotionCoreSecrets.xcconfig` into a worktree first if building there). Ends in `** BUILD SUCCEEDED **`.

- **[gotcha:project]** `WatchMessageKeys.swift` historically had NO `import` line. Anything added there touching Foundation types needs an explicit `import Foundation`, or BOTH main and Watch targets fail to compile.
